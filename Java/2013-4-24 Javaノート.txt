オブジェクト指向
もの　　　　方向づける、中心に据える
オブジェクト指向の場合、中心に据えるが意味としてあっている！

・クラスの中に定義できるもの
フィールド
メソッド
コンストラクタ

User u1 = new User();
User()の()は引数なしのコンストラクタを呼んでいる！

コンストラクタには戻り値はない！

public class StringTest {
	public static void main(String[] args) {
		String s1 = "Hello";
		String s2 = "Hello";
		String s3 = new String("Hello");
		String s4 = new String("Hello");
		
		System.out.println(s1 == s2);
		System.out.println(s3 == s4);
		
		System.out.println(s1.equals(s2));
		System.out.println(s3.equals(s4));
	}
}


s1 == s2は変数の中身を比較
s3.equals(s4)は文字列の比較

基本型と参照型のフィールドを初期化しないでSysoutした場合、
byte, short, int, longは0
double, floatは0.0
booleanはfalse
charは空文字
Stringはnullというか
参照型はnullで初期化される！！
この意味はどのオブジェクトも持っていませんという意味！
しかしint型の配列の場合は0、boolean型の配列の場合はfalse、double型の配列の場合はfalseで初期化される。


ローカル変数は初期化しなければならないが
インスタンスフィールド、クラスフィールドは初期化しなくても暗黙的に初期化される。
